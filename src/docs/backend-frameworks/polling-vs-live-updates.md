# Polling vs Live Updates

## Basics
- **Polling** → Client repeatedly requests updates (inefficient).
- **Live Updates** → WebSockets, SSE, GraphQL Subscriptions.

## Why Choose
- **Polling** → Simpler, works when server can't push.
- **Live Updates** → Real-time UX (chat apps, dashboards).
