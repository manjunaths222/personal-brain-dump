# RDS vs DynamoDB vs MongoDB

## Basics
- **RDS** → SQL relational DB.
- **DynamoDB** → AWS-managed NoSQL key-value store.
- **MongoDB** → NoSQL document store.

## Why Choose
- **RDS** → Complex queries, ACID transactions.
- **DynamoDB** → High-scale, low-latency, serverless NoSQL.
- **MongoDB** → Flexible schema, developer-friendly JSON docs.

## When to Use
- **RDS** → Financial apps, ERP.
- **DynamoDB** → IoT, gaming leaderboards, large-scale APIs.
- **MongoDB** → Content-driven apps, flexible schema use cases.
