# RDS vs Aurora

## Basics
- **RDS** → Managed relational DB (MySQL, Postgres, Oracle, etc.).
- **Aurora** → AWS-built RDS-compatible DB with 5x performance, auto-scaling storage.

## Why Choose
- **RDS** → Simpler, cheaper for standard workloads.
- **Aurora** → High performance, global scaling, fault-tolerant storage.

## When to Use
- **RDS** → Small-medium scale apps.
- **Aurora** → Large SaaS platforms, multi-region availability.

## Interview Q&A
**Q: Why Aurora over MySQL RDS?**
Better scaling, performance, fault tolerance, and integration with AWS.
